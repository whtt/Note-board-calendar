## 🐛 核心 Bug 修复 (Bug Fixes)

### 彻底消灭“幽灵焦点”死锁问题 (Chromium Focus Deadlock)

* **问题表现** ：在 v1.1.0 中，当用户在仪表盘点击“收录诗词”，或在书斋中将笔记“付之一炬”后，会遭遇输入框失焦、键盘失效的“白化”状态，必须将鼠标切出软件再切回（强制触发 OS 级别的 `WM_SETFOCUS`）才能唤醒光标。
* **病理追踪** ：此问题源于 Electron 与 Windows/Mac 操作系统间的焦点抢夺（Chromium Issue 1163602）。原生的 `alert` 和 `confirm` 弹窗会强行劫持系统级焦点 (OS Focus)，在弹窗关闭瞬间若立刻伴随 DOM 的销毁与重建，渲染引擎的焦点池将彻底悬空死锁。
* **终极根治** ：我们在全局架构中 **物理拔除了所有的原生系统弹窗** ，所有交互均回归 Vue 虚拟 DOM 内部完成，从根本上切断了与操作系统底层抢夺焦点的可能。

## ✨ 交互与美学升级 (UI/UX Improvements)

借由本次底层 Bug 修复，我们对案头的交互体验进行了全面重构，使其更符合现代桌面 OS 的标准：

### 1. 引入全局浮动通知引擎 (Zen Toast System)

* **全面封杀 `window.alert`** 。现在的系统反馈不再是冰冷且打断心流的原生弹窗。
* **优雅的视觉反馈** ：无论是“🔖 诗词收录”、“✅ 雅记导入”还是“📁 迁都（更改存储目录）”，系统将在窗口顶部正中，弹出如 macOS 般轻盈的磨砂玻璃浮层（Toast）。伴随古典色调（黛绿/绛红），浮层会在 3 秒后如呼吸般自动消散，全程绝不打断用户的操作心流。

### 2. 重构“付之一炬”内联确认流 (Inline Confirmation)

* **废弃 `window.confirm`** 。将原本割裂的弹窗确认，重构为极其克制的内联 UI。
* 点击“付之一炬”后，操作按钮将在原位平滑展开为 `确认焚毁？ 是 | 否` 的淡红色提示。鼠标无需大范围移动，视觉焦点不发散，确认删除后，新纸瞬间铺开，光标精准待命。

### 3. 书斋“开卷落笔”强化 (Auto-Focus Enhance)

* 优化了底层 `BasePaper` 的生命周期调度。
* 当用户点击新建或焚毁旧卷宗后，底层 Markdown 渲染器会在 100ms 动画结算完毕的瞬间，霸道且精准地将光标 `focus()` 锁定在等宽字符的编辑区内。真正做到“开卷即落笔，心流零延迟”。
